# SkillLand

SkillLand - локальный сайт с профориентацией, мини-играми, аккаунтами, результатами, отзывами и скрытой админ-панелью. Проект запускается без сборки: сервер отдает обычные HTML-файлы, а данные сохраняются в SQLite-базу `skillland.db`.

## Что нужно для запуска

Нужен установленный Node.js.

Проект работает на Node.js 20 и новее. На Node.js 22+ сервер использует встроенный SQLite-модуль `node:sqlite`. На Node.js 20 сервер автоматически использует системную команду `sqlite3`, которая есть на macOS.

Проверить Node.js:

```bash
node -v
```

Если команда не найдена, нужно установить Node.js с официального сайта:

```text
https://nodejs.org
```

## Быстрый запуск

Перейти в папку проекта:

```bash
cd "/Users/nikitasokovyh/Downloads/skillland 2"
```

Запустить скрипт через `zsh`:

```bash
zsh start.sh
```

После запуска сайт откроется в браузере:

```text
http://localhost:3000
```

Скрытая админ-панель:

```text
http://localhost:3000/secret-admin-panel-x7k9
```

Окно терминала нельзя закрывать, пока нужен сайт. Для остановки сервера нажать:

```bash
Ctrl+C
```

## Если `./start.sh` не запускается

Иногда macOS может не дать запустить файл напрямую, даже после `chmod +x`. В таком случае запускай его через `zsh`:

```bash
zsh start.sh
```

Если все-таки нужен запуск именно как `./start.sh`, можно попробовать:

```bash
chmod +x start.sh
xattr -d com.apple.quarantine start.sh 2>/dev/null || true
./start.sh
```

Но для этого проекта достаточно команды `zsh start.sh`.

## Ручной запуск без `start.sh`

Можно запустить сервер напрямую:

```bash
cd "/Users/nikitasokovyh/Downloads/skillland 2"
NODE_NO_WARNINGS=1 node server.js
```

Что делает команда:

- `cd ".../skillland 2"` - переходит в папку проекта.
- `NODE_NO_WARNINGS=1` - скрывает технические предупреждения Node.js.
- `node server.js` - запускает сервер сайта.

## Запуск только на компьютере

Если сайт нужен только на этом компьютере:

```bash
HOST=127.0.0.1 NODE_NO_WARNINGS=1 node server.js
```

Адрес будет:

```text
http://localhost:3000
```

Такой режим не рассчитан на открытие сайта с телефона.

## Запуск для телефона в одной Wi-Fi сети

По умолчанию сервер слушает `0.0.0.0`, поэтому сайт можно открыть с телефона, если телефон и компьютер подключены к одной сети.

Запуск:

```bash
NODE_NO_WARNINGS=1 node server.js
```

На компьютере сайт открывается так:

```text
http://localhost:3000
```

На телефоне нужно открыть IP-адрес компьютера:

```text
http://IP-адрес-компьютера:3000
```

Например:

```text
http://192.168.1.25:3000
```

Узнать IP-адрес на macOS можно командой:

```bash
ipconfig getifaddr en0
```

Если команда ничего не выводит, можно попробовать:

```bash
ipconfig getifaddr en1
```

## Если порт 3000 занят

Скрипт `start.sh` проверяет порт `3000`. Если порт уже занят, он считает, что сервер, возможно, уже запущен, и откроет:

```text
http://localhost:3000
```

Если нужно вручную найти процесс на порту:

```bash
lsof -iTCP:3000 -sTCP:LISTEN
```

Остановить найденный процесс можно по его PID:

```bash
kill PID
```

Где `PID` - номер процесса из вывода команды `lsof`.

## Как работает сайт

Главная страница открывается из файла `index.html`. Сервер отдает ее по адресу:

```text
http://localhost:3000
```

На главной странице есть:

- навигация;
- мобильное меню;
- вход и профиль;
- раздел с отзывами;
- профориентация;
- карта профессий;
- встроенные мини-игры;
- окно с рекомендациями после прохождения игры.

Мини-игры открываются внутри `iframe` на странице `index.html`. Когда игра заканчивается, она отправляет результат в главную страницу через `postMessage`. Главная страница принимает сообщение, сохраняет результат через API и показывает окно с рекомендацией.

Формат сообщения от игры:

```js
{
  type: 'skillland-game-complete',
  game: 'programmer',
  score: 84,
  summary: 'Игра завершена'
}
```

Допустимые значения `game`:

```text
programmer
economist
chef
miner
medic
```

## Аккаунты

Аккаунты сохраняются в таблицу `accounts` в базе `skillland.db`.

Пользователь вводит ник и пароль. Если ник уже есть в базе, сервер проверяет пароль и выполняет вход. Если ника нет, сервер создает новый аккаунт.

API входа и регистрации:

```text
POST /api/auth/login
```

Тело запроса:

```json
{
  "nick": "player",
  "password": "1234"
}
```

Ответ:

```json
{
  "nick": "player",
  "scores": {
    "programmer": 0,
    "economist": 0,
    "chef": 0,
    "miner": 0,
    "medic": 0
  }
}
```

Важно: сейчас пароли сохраняются в базе как обычный текст. Для локального учебного проекта это работает, но для публикации в интернете нужно добавить хеширование паролей.

## Профиль

Профиль показывает текущего пользователя, результаты по профессиям и самое подходящее направление.

Получить профиль:

```text
GET /api/auth/profile?nick=player
```

Изменить ник или пароль:

```text
PUT /api/auth/profile
```

Тело запроса:

```json
{
  "nick": "oldNick",
  "newNick": "newNick",
  "newPassword": "newPassword"
}
```

Если меняется ник, сервер также переименовывает связанные результаты и отзывы.

## Результаты игр

Результаты сохраняются в таблицу `scores`.

API сохранения результата:

```text
POST /api/scores
```

Тело запроса:

```json
{
  "nick": "player",
  "game": "programmer",
  "score": 84
}
```

Особенности:

- результат ограничивается диапазоном от `0` до `100`;
- если по этой игре уже был результат, сохраняется лучший результат;
- для одной пары `nick + game` в базе хранится одна запись.

## Отзывы

Отзывы сохраняются в таблицу `reviews`.

Получить все отзывы:

```text
GET /api/reviews
```

Создать отзыв:

```text
POST /api/reviews
```

Тело запроса:

```json
{
  "nick": "player",
  "game": "programmer",
  "stars": 5,
  "text": "Понравилась игра"
}
```

Ограничения:

- один пользователь может оставить только один отзыв на одну игру;
- оценка ограничена от `1` до `5`;
- текст ограничен 500 символами;
- ник в отзыве ограничен 32 символами.

## Скрытая админ-панель

Админ-панель не показывается в обычной навигации сайта. Она открывается только по прямому адресу:

```text
http://localhost:3000/secret-admin-panel-x7k9
```

На телефоне или другом устройстве в сети:

```text
http://IP-адрес-компьютера:3000/secret-admin-panel-x7k9
```

В админке можно смотреть:

- количество аккаунтов;
- количество игроков с результатами;
- средний результат;
- количество сыгранных игр;
- количество отзывов;
- список аккаунтов;
- список результатов;
- список отзывов.

Также есть действия:

- удалить аккаунт;
- удалить отзыв.

При удалении аккаунта сервер удаляет связанные результаты и отзывы этого пользователя.

API админки:

```text
GET /api/admin/stats
DELETE /api/admin/delete-account
DELETE /api/admin/delete-review
```

Удалить аккаунт:

```json
{
  "nick": "player"
}
```

Удалить отзыв:

```json
{
  "id": 1
}
```

Важно: сейчас админ-панель скрыта ссылкой, но не защищена паролем. Для публикации в интернете нужно добавить настоящую авторизацию администратора.

## База данных

База лежит в файле:

```text
skillland.db
```

Она создается автоматически при запуске `server.js`, если файла еще нет.

Таблицы:

```text
accounts
scores
reviews
```

`accounts` хранит:

- `id` - номер аккаунта;
- `nick` - ник пользователя;
- `password` - пароль;
- `created_at` - дата создания.

`scores` хранит:

- `id` - номер результата;
- `nick` - ник пользователя;
- `game` - ключ игры;
- `score` - лучший результат;
- `updated_at` - дата обновления.

`reviews` хранит:

- `id` - номер отзыва;
- `nick` - ник пользователя;
- `game` - ключ игры;
- `stars` - оценка;
- `text` - текст отзыва;
- `date` - дата для показа на сайте;
- `created_at` - дата создания в базе.

## Миграция старых отзывов

Если рядом с `server.js` есть файл `reviews.json`, сервер при запуске переносит отзывы в SQLite и переименовывает файл:

```text
reviews.json -> reviews.json.migrated
```

Это сделано, чтобы старые отзывы не потерялись и больше не хранились в JSON-файле.

## Локальные данные в браузере

Раньше аккаунты могли храниться в `localStorage`. Сейчас сайт использует базу данных.

В браузере остается только текущая сессия пользователя:

```text
skillland_session_v1
```

Также есть отметка о переносе старых локальных аккаунтов:

```text
skillland_db_migrated_v1
```

Если в браузере были старые локальные аккаунты, сайт пытается перенести их в базу при открытии.

## Частые проблемы

### `node:sqlite` не найден

На Node.js 20 это нормально. Сервер должен автоматически переключиться на системный `sqlite3`.

Проверить:

```bash
node -v
```

Если после исправления сервер все равно падает с ошибкой `node:sqlite`, значит используется старая версия `server.js`. Обнови файл и запусти:

```bash
NODE_NO_WARNINGS=1 node server.js
```

### Сайт не открывается

Проверить, запущен ли сервер:

```bash
lsof -iTCP:3000 -sTCP:LISTEN
```

Если сервер не запущен, выполнить:

```bash
zsh start.sh
```

### Телефон не открывает сайт

Проверить:

- телефон и компьютер в одной Wi-Fi сети;
- открыт адрес с IP компьютера, а не `localhost`;
- сервер запущен без `HOST=127.0.0.1`;
- macOS или антивирус не блокирует входящие подключения.

Правильный формат адреса для телефона:

```text
http://192.168.x.x:3000
```

### Админка не открывается

Проверить точный адрес:

```text
http://localhost:3000/secret-admin-panel-x7k9
```

Если открыть просто `admin.html`, часть функций может работать неправильно, потому что данные админки приходят через серверное API.
